# Bibliogon v0.40.0 — 2026-05-29

A large authoring-depth release across the comic-book and picture-book surfaces plus an articles content-model expansion. ~100 commits since v0.39.0 (210 files, +29390 / -5602) across five coordinated streams: Content-Types SSoT, Publication-status parity, in-place title editing, comic-book depth (bubble overhaul + multi-panel layouts + panel reorder/move), and picture-book layout expansion (Phases 1-3). Plus a Settings-Completeness batch and a shared FullscreenButton.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

## Download

| Platform | File |
|----------|------|
| Windows | `bibliogon-launcher.exe` |
| macOS (Apple silicon) | `bibliogon-launcher-macos.zip` |
| Linux | `bibliogon-launcher-linux` (ELF binary) |

Each platform also ships a `*.sha256` checksum next to the binary.

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

```powershell
# Windows
Get-FileHash -Algorithm SHA256 .\bibliogon-launcher.exe
Get-Content .\bibliogon-launcher.exe.sha256
```

The hashes must match.

If your operating system warns about an unsigned binary, see the [Bibliogon installation overview](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/installation.md).

## Action required

**None for existing users.** v0.40.0 ships two auto-applying schema migrations: a backfill of `Article.content_type == "article"` → `"blogpost"` plus a new nullable `article_metadata` column, and a new `Book.status` column backfilled to `"draft"`. Both apply on first start with no data loss. All existing articles become Blog posts; all existing books become Drafts.

## What's new

### Content-Types SSoT (8 article types)

The long-reserved `Article.content_type` column is repurposed as an 8-type discriminator per its original model intent.

- **8 types** from a new `backend/config/content-types.yaml` registry: Blog post (default), Tutorial, Review, Essay, Newsletter, Interview, Listicle, Short story. `GET /api/content-types` serves the registry; a drift-detector test keeps the Pydantic `ContentType` Literal in sync with the YAML.
- **Per-type extra fields** (tutorial difficulty / prerequisites / duration, review work / author / rating, newsletter issue / send date, interview partner / role) stored in a new `article_metadata` JSON column; surfaced in the ArticleEditor metadata sidebar.
- **AD split-button** (`SplitButton` primitive, RCU 2-surface with the Book Dashboard): default click creates a Blog post; the chevron exposes the other 7 types. Content-type badge in the AD card + list-row footers.
- **i18n** for all 8 types + per-type field labels in 8 catalogs. Help-doc pair `articles/content-types`.

Backlog: `CONTENT-TYPES-SSOT-01` (renamed mid-arc from `ARTICLE-TYPES-SSOT-01`).

### Publication-status parity for Books

Book gains the Article publication lifecycle (draft / ready / published / archived) behind a shared `PublicationStatus` enum + `_PUBLISHING_LIFECYCLE` tuple, so a future status change propagates to both surfaces in one edit. BookCard + BookListView gain a status badge. `ArticleStatus` becomes a back-compat alias. Foundation for the title-editing published-work warning.

Backlog: `PUBLICATION-STATUS-BOOK-PARITY-01`.

### In-place title editing across all editors

A shared `EditableTitle` component (click-the-pencil inline edit; Enter/blur saves, Escape reverts) on all four editor surfaces: Article, prose Book (chapter sidebar), picture-book PageEditor, comic-book ComicBookEditor. Published or archived works gate the edit behind a yellow warning banner with an acknowledgment button (a title change must be carried over to the publishing platform manually); drafts edit directly. i18n in 8 catalogs; help-doc pair `editor/title-editing`.

### Comic-book depth

- **Bubble overhaul** — all 6 bubble types (speech, thought, narration, shout, whisper, sound-effect) render as a single continuous SVG path (outline + tail in one shape), with the same path generator running in the editor preview and the WeasyPrint PDF walker. Per-type tail behaviour (thought circle-chain, shout spike-absorption, narration forced no-tail). Pointer-drag to reposition a bubble and drag its tail tip. a11y (button-name + contrast) pass. Help-doc pair `books/comic-bubbles` + visual-regression baselines.
- **Multi-panel grid layouts** — 7 grid templates (single_panel through grid_3x3) selectable per page via `ComicGridTemplatePicker`; Add-Panel disables at the template's cell capacity. An overflow handler offers Move-to-new-pages / Delete / Cancel when a layout switch would exceed capacity.
- **Panel reorder + cross-page move** (`COMIC-PANEL-CROSS-PAGE-MOVE-01`) — same-page reorder via a dnd-kit drag handle on each panel (new bulk `POST .../comic-pages/{page_id}/panels/reorder` endpoint, two-phase position update). Cross-page move via a "Move to page" action menu listing other pages with capacity (`Seite N - count/max Panels`); full pages disabled with a `(voll)` hint; reuses the `ComicPanelUpdate.page_id` PATCH and re-normalises the source page. Help-doc pair `books/comic-panels`.

### Picture-book layout expansion (Phases 1-3)

New `PageLayout` variants beyond one-image-per-page: split_horizontal, split_vertical, two_images_text_center, image_border_text_center (Phases 1-2), and a free-form **collage** layout (Phase 3) with drag-positioned image + text regions (`useDragPosition` hook), z-index ordering, and a matching collage PDF walker. LayoutPicker groups layouts into categories; i18n for all new layouts in 8 catalogs.

### Settings-Completeness batch + shared FullscreenButton

- Backup-history per-entry delete + clear-all (new DELETE endpoints + Settings UI).
- `white_label` feature flag gating the Erweitert tab.
- Per-field `X` clear button extracted (RCU) across 4 search surfaces.
- KDP wizard default marketplace + Picture-Book PDF defaults migrated from localStorage to `app.yaml`.
- Confirmation-skip mode for non-destructive dialogs; Autoren tab profile-vs-database clarification; persisted Tier1/Tier2 collapsible open-state per surface.
- Shared `FullscreenButton` adopted across the page editors (picture-book + comic-book + storyboard). New Settings > Backups help page (DE + EN).

## Fixed

- Comic bubble position mismatch between editor and PDF export (SVG explicit width/height; CSS `inset` shorthand WeasyPrint couldn't parse).
- Comic bubble text reads as solid black by default.
- Picture-book secondary-image upload affordance now visible.
- Locale-aware date formatting across 8 surfaces.
- Fullscreen exit tooltip mentions both Esc and F11.

## Compatibility

- **`Article.content_type`**: legacy `"article"` rows auto-backfill to `"blogpost"` on migration AND on .bgb restore. New `article_metadata` round-trips through backups. The `non_article` 422 facet on book-from-articles is dropped — every row in the articles table is by definition an article.
- **`Book.status`**: new column, backfilled `"draft"`. Optional on create/PATCH.
- **Picture-book + comic layouts**: region geometry persists through the existing `Page.layout_config` namespaced JSON — no new columns.
- **i18n**: all new surfaces added across 8 catalogs; the parity test is green.
