## [0.23.0] - 2026-04-25

Major workflow milestone for self-publishing authors maintaining multi-language books in external git hosting. Plugin-git-sync ships its full PGS-02..05 rollout: bi-directional sync to a remote write-book-template repo, three-way smart-merge on re-import with a per-chapter conflict UI, branch-driven multi-language detection with auto-linked translation groups, and a unified-commit bridge to core git so authors who run both subsystems on the same book can commit everywhere in one click. PGS-01 (the import-only MVP that landed before v0.22.0) is the foundation the four new phases build on.

The release also surfaces the post-v0.22.0 backend/UI polish that shipped through v0.22.1: multi-book BGB import on an XState v5 state graph, sticky action-button footers across 13 dialog modals, the EnhancedTextarea wrapper (CSS lowlight + Markdown/HTML preview + fullscreen + copy-to-clipboard) on every metadata textarea, and a structured error-reporting path (WizardErrorBoundary + Copy details + Report on GitHub).

### Action required for v0.22.0 users

If you reached v0.22.0 via `alembic upgrade head` rather than a fresh install, run it again after pulling v0.23.0:

```bash
cd backend && poetry run alembic upgrade head
```

The migration `c6d7e8f9a0b1` (added in v0.22.1) backfills `books.tts_speed`; v0.23.0 layers two more on top (`d7e8f9a0b1c2` for `git_sync_mappings`, `e8f9a0b1c2d3` for `books.translation_group_id`). All migrations are idempotent.

### Known limitation

PAT-via-UI is **not yet wired** for either core git or plugin-git-sync push. Pushing to a remote requires ambient credentials at the OS level: the user's SSH agent / `~/.ssh/id_*` for SSH URLs, or the system git credential helper for HTTPS. PAT integration through Bibliogon's own credential store is the next git-sync follow-up.

### Plugin-git-sync rollout

**PGS-01 (foundation, shipped pre-v0.22.0).** Plugin scaffold, `RemoteSourceHandler` protocol, `POST /api/import/detect/git`, GitImportHandler that clones into the orchestrator's staging dir and delegates to the core WBT importer. Wizard Step 1 gains a git URL input. Public HTTPS only at this stage; auth + branch selection deferred. Recapped here because the four new phases stand on it.

**PGS-02 - Commit to Repo.** New `git_sync_mappings` table persists the post-import clone into `uploads/git-sync/{book_id}/repo/`. `commit_to_repo` re-scaffolds the book via plugin-export's `scaffold_project`, replaces the working tree (preserves `.git/`), and creates one commit. Push uses ambient git credentials; failures surface as typed `PushFailedError` with stable `.reason` slugs the router maps to specific status codes. UI: GitSyncDialog with mapping snapshot, dirty-warning, optional message + push toggle, last-commit confirmation. Sidebar "Sync zum Repo" button shows up when the book has a mapping.

**PGS-03 - Smart-Merge on re-import.** Three-way diff service reads base + remote WBT chapters at arbitrary git refs via `git ls-tree` + `git show` (no working-tree checkout) and converts the local DB through `tiptap_to_markdown` so the diff sees what commit-to-repo would write. Identity is `(section, slug-of-title)`. Eight classifications cover every case (unchanged / remote_changed / local_changed / both_changed / remote_added / local_added / remote_removed / local_removed) plus the same-edit-on-both-sides not-a-conflict case. Endpoints `POST /diff` and `POST /resolve`; the resolve path advances `last_imported_commit_sha` so the next diff stops re-reporting the same changes. UI: GitSyncDiffDialog lists actionable rows with smart per-row defaults (remote_changed -> Take Repo, conflict -> Keep Bibliogon).

**PGS-04 - Multi-language branch linking.** New `books.translation_group_id` column. Branch-driven language detection: `main-XX` -> `XX`; bare `main` -> `metadata.yaml.language`. Locale tags like `main-de-AT` are explicitly rejected so the suffix stays unambiguous. `import_translation_group` clones once, enumerates every matching branch, runs the WBT importer per checkout, persists a per-book clone with its own `GitSyncMapping`, and links the resulting books with one shared id. Manual link/unlink for separately-imported books via a Settings-style picker on the metadata editor. Flat cross-link model - no master/translation hierarchy.

**PGS-05 - Core-git bridge with unified commit.** Per-book `threading.Lock` with a 30 s default timeout. `unified_commit` runs core git first (smaller blast radius), plugin-git-sync second; per-subsystem failures land in the response payload (`status: ok | skipped | nothing_to_commit | failed`) rather than as a single hard 500. UI: a banner + primary "Commit ueberall" button next to the single-subsystem button when both core git and plugin-git-sync are active. Per-subsystem outcomes render as a status-coded result list under the form.

### Other major work since v0.22.0

**Multi-book BGB import.** `.bgb` archives carrying multiple books now render a per-book selection list with bulk select-all/deselect-all, per-row duplicate handling (Skip / Overwrite / Create-new), and chapter/cover badges. Backend extends `DetectedProject` with `is_multi_book` + `books: list[DetectedBookSummary]`; the orchestrator dispatches to `execute_multi` on multi-book detect. New `wizardMachine` (XState v5; 7 states + 3 guards) acts as the testable data layer.

**Sticky modal action-button footers.** Action buttons in long-content modals (ImportWizard, ChapterTemplate, CreateBook, ErrorReport, Export, SaveAsTemplate) now stay visible via a global `.dialog-content-wide .dialog-footer` sticky rule. BackupCompare and GitBackup got bespoke inline sticky on their long-content surfaces. 13 modals audited end-to-end.

**EnhancedTextarea.** Universal textarea wrapper with toolbar (copy, word/char counter, autosize, fullscreen) and tab-switchable preview for `css` (lowlight syntax), `markdown` (react-markdown + remark-gfm) and `html` (DOMPurify-sanitized) fields. All metadata textareas migrated.

**Structured wizard error reporting.** `WizardErrorBoundary` catches render exceptions inside the import wizard. Rewritten `ErrorStep` exposes Copy details (clipboard-ready markdown bundle) plus Report Issue (pre-filled GitHub Issues URL). Replaces the prior black-hole UX where a failed import left the modal cratered with no actionable message.

**Three-option author picker.** Wizard now handles `.bgp`/`.bgb` with no author by offering: create-new, pick-existing, defer. Defer is gated behind a Settings toggle `Allow books without author` (default OFF). Migration `b5c6d7e8f9a0` makes `Book.author` nullable.
