# Bibliogon v0.26.6 — 2026-05-04

Hotfix for v0.26.5 Windows release-event failure. v0.26.5's `.gitattributes` fix was correct hygiene but not the actual cause; v0.26.6 fixes the real bug. Windows users should install v0.26.6.

## What broke

`sync_versions.py:regenerate_install_sh` invoked the bash helper via `subprocess.run(["bash", str(script), "--check"])`. On Windows GitHub Actions runners, PATH ordering means subprocess can resolve `bash` to the WSL2 launcher (`C:\Windows\System32\bash.exe`) instead of Git Bash (`C:\Program Files\Git\bin\bash.exe`). WSL2 mishandles native Windows paths passed as args; the bash script never ran correctly; sync_versions.py reported spurious drift.

## Fix

Reimplemented the install.sh `--check` and regenerate logic in pure Python inside `sync_versions.py`. Reads `install.sh.template`, substitutes `@@BIBLIOGON_VERSION@@` from the canonical pyproject, compares to `install.sh` content. No bash subprocess, no PATH-dependent resolution, no cross-platform path translation.

`generate_install_sh.sh` stays as the canonical human-facing CLI tool; the Python implementation is the cross-platform equivalent for tooling that does not want to spawn subprocesses.
