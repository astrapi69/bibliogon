# Bibliogon v0.47.0 — 2026-06-05

The **full offline PWA** release. The static GitHub Pages build now boots with no backend, loads settings / i18n / type registries from a seeded IndexedDB, supports full prose-book and article authoring offline, and fires **zero `/api` network requests** (a `guardedFetch()` choke point enforces it, with an E2E hard gate). Plus the GitHub Pages deploy split (app + docs on separate sites), a non-toggling single-line menu header, and a settings-persistence fix.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

```bash
cat bibliogon-launcher-<platform>.sha256
```

## Action required

**None.** This release ships no schema migrations. Existing data, backups (`.bgb`) and projects (`.bgp`) are unaffected.

## What's new

### Added

- **Full offline PWA.** The static GitHub Pages build (`astrapi69.github.io/bibliogon/`) boots with no backend at all. It loads default settings, the 8 i18n catalogs, book-types, content-types and the plugin list from a seeded IndexedDB, and supports creating, editing, deleting and reloading prose books and articles entirely offline. Backend-only features (export, Git, Medium import, LAN, audiobook, story bible, picture-book / comic editors, danger-zone backup, AI) are gated in offline mode with a translated "requires the desktop app" hint instead of a broken button.
- **Seed pipeline.** `make generate-seed-data` (`scripts/generate-seed-data.py`) reads the canonical backend YAML single-source-of-truth and emits committed JSON under `frontend/src/storage/seed/` (i18n catalogs, settings defaults, book/content types, plugin metadata) whose shapes mirror the `/api` responses. `DexieStorage` populates its reference tables on first init via an idempotent `ensureSeeded()`. Re-run and commit the JSON whenever a backend catalog, `app.yaml` default, or type registry changes.
- **GitHub Pages deploy split.** The app deploys to `astrapi69.github.io/bibliogon/` and the docs to `astrapi69.github.io/bibliogondocs/`, with a `404.html` SPA redirect for deep links. Both workflows auto-trigger on push to `main`.

### Fixed

- **Menu single-line header.** The top menu no longer flips between the full bar and the hamburger when the language or default content-type changes. It now switches on a fixed viewport breakpoint (`--breakpoint-menu`, 1200px) measured against the worst-case label width across all 8 locales: hamburger below 1200px, full bar above.
- **Settings "skip non-destructive confirmations" persistence.** The `behavior.skip_non_destructive_confirmations` toggle was silently dropped on save because the settings update schema had no `behavior` field. It now persists.

### Changed

- **`guardedFetch()` single egress.** Every network call in the client — the JSON `request()` helper, the raw upload / blob / export calls, and the import wizard — now flows through one `guardedFetch()` choke point. On the backendless offline build it rejects before any fetch, so the offline build fires zero `/api` requests regardless of which `api.*` method is called, and any future call that forgets the storage seam is auto-covered. An offline E2E installs a global `route.abort()` on `**/api/**` as a hard regression gate.
- **`IStorageService` seam extended.** The storage seam now covers settings, the 8 i18n catalogs, book-types, content-types and writing-sessions in addition to books / chapters / articles, with both `ApiStorage` (online) and `DexieStorage` (offline) implementations. Call sites read and write through `getStorage()`, a no-op in online mode and IndexedDB-backed offline.
- **Documentation discipline.** New `.claude/rules/code-hygiene.md` rule: no inline comments; code is self-explanatory and explanations live in docstrings / TSDoc. The recent offline-PWA code was cleaned up accordingly.
