# Bibliogon v0.42.0 — 2026-05-30

The Story Bible release. A new `plugin-story-bible` gives every book a per-book database of fiction-writing entities (character / setting / plot point / item / lore), and a deep Story Bible ↔ Storyboard integration turns it into Bibliogon's standout feature: while planning a story visually, the author sees which entities appear on which pages, links them by drag-and-drop, reviews character arcs on a visual timeline, gets advisory continuity warnings, and exports the whole bible to share. Plus a content-type-neutrality pass, a component-consistency + accessibility sweep, and two safety/stability fixes.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

## Action required

**None.** The new Story Bible tables (`story_entities`,
`story_entity_page_links`) are created automatically on first start
(the FastAPI lifespan runs the Alembic migrations). Existing data,
backups (`.bgb`) and projects (`.bgp`) are unaffected.

## What's new

### Added

- **Story Bible plugin** — a per-book database of fiction entities
  (character / setting / plot point / item / lore) with per-type
  metadata, rich-text descriptions, a colored sidebar, a detail/edit
  view, and CRUD under `/api/story-bible/...`. Gated behind plugin
  activation.
- **Story Bible ↔ Storyboard integration** — link entities to pages
  (or chapters), see entity badges on Storyboard cards, drag an entity
  onto a card to link it, track every appearance from the entity's
  detail view, filter the Storyboard to pages where selected entities
  co-appear, review a visual **Arc View** of character arcs across the
  book, get advisory **continuity** warnings, and **export** the bible
  to Markdown.
- **Storyboard for comic books** — the Storyboard now opens from the
  comic-book editor too (previously picture-book only).
- **Per-content-type default titles** — new content types created from
  the Article-Dashboard split-button get a matching default title
  (Neuer Blogpost / Neues Tutorial / …).

### Changed

- **Content-type neutrality** — the article editor spans 8 content
  types, so user-facing "Artikel/Article" became the neutral "Text",
  and the type dropdown label became "Textart" / "Content type".
- **Component-consistency + accessibility sweep** — buttons, selects,
  inputs, checkboxes, sliders, badges and cards unified onto global
  CSS classes + tokens; WCAG hardening across editor surfaces.
- Story Bible toggle moved into the chapter sidebar's actions.

### Fixed

- KDP check-metadata accepts `keywords` as a list (Publishing Wizard
  422).
- Story Bible toggle no longer overlaps the editor toolbar.

### Security

- **Zip Slip (CWE-22)** guard on all archive-extraction paths (`.bgb`
  backup import, backup compare, project import) — a crafted archive
  can no longer write files outside the extraction sandbox.
- **App-wide React error boundaries** — a render crash in one screen
  now shows a friendly reload prompt instead of blanking the app.

Backend pytest 2399 → 2442 (+43, 1 skipped); Vitest 2487 → 2549 (+62).
