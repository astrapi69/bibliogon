# Bibliogon v0.35.1 — 2026-05-18

Fast-follow patch for the v0.35.0 donation-visibility gap. The S-03 reminder banner shipped in v0.19.0 had been functionally invisible to new users because the 90-day grace gate meant no banner fired until day 91. Bibliogon's donation-based sustainability model needs visibility from week one, not month four.

## Action required

None. Patch bump; no schema migrations, no API changes. Existing users with installs older than 7 days will now see the donation reminder banner at the top of the app (was: invisible until day 91); dismiss via Support / Not now / X / Escape to set the cooldown.

## What's new

### Donation-visibility fix

The S-03 donation reminder banner has been in production since v0.19.0 but was functionally invisible to new users because of a 90-day grace gate. v0.35.1 fixes three things:

**1. Grace period 90 → 7 days.** New users see the reminder after a one-week settling-in window instead of after three months.

**2. App-level mount.** Banner now appears at the top of every page (Dashboard, Book Editor, Article Editor, Settings, Help) and persists across navigation until the user actively dismisses. Pre-v0.35.1 it only mounted on the Dashboard, so users in active workflows never saw it.

**3. Accessibility.** Escape keyboard dismiss (behaves like Not-now → 90-day cooldown). `aria-live="polite"` on the banner root so screen readers announce the reminder without stealing focus.

### Industry-research synthesis (for the cooldown values)

The cooldowns unchanged at 90 days dismissed / 180 days donated. Reference points across donation-based open-source / non-profit projects:

| Source | Dismissed cooldown | Donated cooldown |
|--------|--------------------|-----------------|
| Wikipedia/Wikimedia | Annual drive only | 365 days |
| Mozilla Foundation | No persistent banner | 90 days email |
| Mastodon (per-server) | 30-90 days | varies |
| Signal Foundation | 90+ days during drive | 365 days |
| Blender Foundation | 30-90 days non-donor | 180-365 days donor |

Bibliogon's 90 / 180 sits in the median band. Preserves donor-respect (6-month silence after a donation click-through) while keeping awareness for non-donors (re-show every 3 months).

## Why a patch and not a minor?

This is a bug-class fix (banner gate too tight) + a UX adjustment (App-level mount), not a new feature. All three changes are backward-compatible: existing users keep their localStorage state; existing cooldowns honored as before; the only behavior change is that pre-Bibliogon-day-91 installs now see the banner where they didn't before.
