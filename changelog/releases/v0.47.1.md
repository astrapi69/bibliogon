# Bibliogon v0.47.1 — 2026-06-05

Patch release fixing the offline PWA's first-render data loading: the book-type and content-type dropdowns (and registries) could come up empty in Dexie mode.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

## Download

| Platform | File |
|----------|------|
| Windows | `bibliogon-launcher.exe` |
| macOS (Apple silicon) | `bibliogon-launcher-macos.zip` |
| Linux | `bibliogon-launcher-linux` (ELF binary) |

Each platform also ships a `*.sha256` checksum next to the binary.

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

## Action required

**None.** This release ships no schema migrations. Existing data, backups (`.bgb`) and projects (`.bgp`) are unaffected. The desktop / Docker app is functionally unchanged — this patch only affects the backendless offline (GitHub Pages) build.

## What's new

### Fixed

- **Offline PWA: empty book/content-type dropdowns.** A DexieStorage lazy-loading race left the default book-type and content-type dropdowns (Settings > Verhalten), and the underlying registries, empty on first render in Dexie mode. The one-shot registry / i18n / settings providers ran before the lazy IndexedDB backend finished loading, fell back to the (offline-rejected) API client, and never retried. DexieStorage is now preloaded before the first render when an explicit dexie mode is set. (#32)
- **Offline PWA: stray `/api/health` request.** `verifyBackendVersion()` fired a raw `/api/health` request even on the backendless build; it is now skipped in forced-offline (dexie) mode.
- **E2E robustness.** Fixed three offline-spec bugs surfaced by the above: the `**/api/**` route glob aborted the Vite-served `/src/api/client.ts` source module, the create-book assertion expected the editor URL (prose books return to the dashboard), and a strict-mode double-match on the optimistic placeholder card.
