# Bibliogon v0.45.0 — 2026-06-02

The QA-hardening + native-i18n release. An adversarial security + data-integrity pass closed every open finding from the v0.44.0 QA report, an accessibility sweep gave every icon-only control an accessible name, and the six auto-translated locale catalogs received a full native-quality translation pass.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

## Download

| Platform | File |
|----------|------|
| Windows | `bibliogon-launcher.exe` |
| macOS (Apple silicon) | `bibliogon-launcher-macos.zip` |
| Linux | `bibliogon-launcher-linux` (ELF binary) |

Each platform also ships a `*.sha256` checksum next to the binary.

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

## Action required

**None.** This release ships no schema migrations. Existing data, backups (`.bgb`) and projects (`.bgp`) are unaffected. Backups now use manifest **v3.0**, which is backward-compatible with v1.0 / v2.0 archives, so older backups still restore.

## What's new

### Security
- **Upload filename path traversal (CWE-22) fixed** at all three upload sites (book assets, article assets, KDP cover validation). A crafted multipart `filename` can no longer write outside the upload root.
- **Archive extraction hardened** — every archive-extraction site now routes through the `safe_extractall()` Zip-Slip guard.
- **Danger Zone reset token is genuinely one-time** — consumed on verify via a nonce store; replayed tokens are rejected.

### Data integrity
- **Backup `.bgb` completeness overhaul** — the backup serializer is now introspection-driven and covers **all 23 content models** (comments, Story Bible links, comic panels, etc.), closing a gap where restoring a backup could silently drop rows. Pinned by a full round-trip test (maximal graph → export → wipe → import → field-level equality).
- Comic cross-page panel move now enforces the destination capacity gate server-side; @-mentions degrade to plain text when their entity is deleted; writing-goal daily progress can no longer go negative; comic bubble anchors are validated to `[0,100]`; the Story-Bible entity-page link enforces its page/chapter XOR via a DB constraint; titles are length- and control-char-validated; the continuity-checker N+1 query is resolved.

### Accessibility
- Every icon-only button across the Dashboard, BookCard, AudiobookPlayer and BookEditor now carries an `aria-label`, and raw Radix dialogs gained the required description. The Dashboard reports **zero axe-core critical/serious violations** on load.

### Native-quality translations
- A full native-quality translation sweep across the six auto-translated locales — Spanish (Latin-American neutral), French (Metropolitan, with French typography), Greek (monotonic Modern Greek), Portuguese (European pt-PT), Turkish (vowel-harmony correct), and Japanese (polite form). Every previously-untranslated UI string is now localized; loanwords and brand names are kept verbatim.
