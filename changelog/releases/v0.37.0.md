## Highlights

### Accessibility — WCAG 2.1 AA

7-commit audit-driven remediation closes the zero-coverage gaps the audit identified:

- **Skip-to-content link** lands on `<main id="main-content">` from every routed page.
- **Landmark sweep** — `<main>` / `<aside>` / `<header>` / `<footer>` on all 8 routed pages; heading hierarchy normalised.
- **Universal `*:focus-visible`** keyboard-focus indicator covering every interactive element + KDP Publishing Wizard step-change focus management.
- **ARIA sweep** — `WizardShell aria-current="step"` on the active step indicator; `aria-pressed` on 20+ Toolbar formatting toggles.
- **TipTap editor accessible name** — `aria-label` + `role="textbox"` + `aria-multiline` on chapter + article + picture-book RichTextEditor surfaces.
- **WCAG AA color-contrast** — 30 axe-core findings resolved by darkening `--text-muted` + `--accent` across all 10 theme variants (5 themes × light/dark). User-adjudicated fix-all path.
- **`@axe-core/react`** devDependency-only via `import.meta.env.DEV` dynamic import. Prod bundle verified clean.
- **Universal `prefers-reduced-motion`** rule coercing all ~42 transition/animation sites to 0.01ms when the OS-level preference is set.

### Danger Zone full-system reset

`Settings → Gefahrenzone / Danger Zone` ships a deliberate three-step wipe:

1. User opens the dialog → frontend hits `POST /api/system/reset/prepare` to receive a per-process random-secret-signed HMAC token (valid 5 minutes).
2. User types the literal string `"RESET"` into a confirmation input.
3. User clicks the destructive action → frontend hits `POST /api/system/reset` with the token + confirmation phrase.

Backend `reset_service.run_reset` cancels in-flight SSE jobs, truncates all 21 tables via `reversed(Base.metadata.sorted_tables)`, re-seeds 5 book + 4 chapter built-in templates, wipes `<data_dir>/uploads/`, `tmp/`, `backup_history.json`, `config/app.yaml`, `config/plugins/*.yaml`, `plugins/installed/*`, and `<config_dir>/secrets.yaml`. Preserves the production-marker tripwire, platformdirs migration breadcrumb, launcher install metadata, and project-tree `licenses.json` (intentionally — those are install-state, not user-state).

Post-reset cleanup on the frontend: `localStorage.clear` + `sessionStorage.clear` + Dexie `BibliogonDB` drop + redirect to Dashboard.

The existing `/api/test/reset` debug-only endpoint stays untouched — both surfaces coexist with different security contracts.

### Dashboard pagination

Book + Article dashboards now paginate via a "Mehr laden" button + a 10/25/50/100 page-size dropdown. Backend `limit` query param on `GET /api/books` + `GET /api/articles` (`ge=1, le=1000`); default `None` preserves the historical no-cap behaviour for non-dashboard callers (BookEditor navigation, backup). PATCH `/api/settings/app` enum-validates `ui.dashboard.{books,articles}_page_size` against `{10, 25, 50, 100}`.

**Selection semantics preserved** — select-all still operates on the full filtered set, not just the visible page. Pagination is a render-cost guard, not a selection boundary.

### Editor display settings

Per-device localStorage-backed preferences for text width (narrow=680px / medium=780px / wide=900px / full), font family (serif / sans / mono), font size (small / medium / large), line height (compact / normal / relaxed). Toolbar popover trigger from the shared Editor.tsx surface — single mount covers BOTH chapter-based BookEditor AND ArticleEditor.

CSS variables on `document.documentElement` cascade into `.tiptap-editor`; `var()` fallbacks match pre-feature literals so opt-out users see no change.

### Book.repository_url metadata field

Optional `Book.repository_url` `String(2000)` nullable column for authors who track their book project in a git repository outside plugin-git-sync. Surfaces in BookMetadataEditor's General tab via a new `RepositoryUrlField` sub-component with **git-sync read-precedence**: when the book is plugin-git-sync-managed (`GitSyncMapping` exists), the field renders read-only with the canonical mapping URL + a "managed by git-sync" hint; otherwise free input editing `Book.repository_url`. Two storage shapes for two genuinely different lifecycles.

### Bulk-restore parity

Single-round-trip endpoints `POST /api/articles/trash/bulk-restore` + `POST /api/books/trash/bulk-restore` accept a list of trash IDs and return a `{restored: [...], failed: [...]}` shape. Replaces the previous "loop the single-restore endpoint per ID" pattern that the Undo toast triggered after bulk-trash. Frontend `Undo` action now hits the bulk endpoint once.

### Medium-import progress polish

Three small but real ergonomic fixes:

- Progress panel sits **above** the preview table during import (was below — easy to miss).
- Action buttons (Run / Cancel / Run-in-background) at the **top** of the preview panel (was bottom — required scroll on long imports).
- `scripts/retro_fix_medium_codeblock_newlines.py` — one-off script to repair collapsed code-block newlines on already-imported legacy rows. Documented in the archive; will not run automatically.
