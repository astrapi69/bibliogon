# Bibliogon v0.35.0 — 2026-05-18

The second Picture-Book release. TipTap rich-text editing for 3 of the 5 picture-book layouts, printable picture-book PDF export with KDP-grade font embedding, 5 OFL fonts curated for children's typography, plus a polished speech-bubble (9-position anchor + width + height + opacity). The medium-import async path also lands (SSE-driven progress + F5-recovery + Run-in-background dock). And the release-automation audit shipped its four scopes — v0.35.0 is the first release cut entirely through the new automation pipeline.

47 commits since v0.34.1. +13718 / -453 lines across 83 files.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser.

## Download

| Platform | File |
|----------|------|
| Windows | `bibliogon-launcher.exe` |
| macOS (Apple silicon) | `bibliogon-launcher-macos.zip` |
| Linux | `bibliogon-launcher-linux` (ELF binary) |

Each platform also ships a `*.sha256` checksum next to the binary.

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

```powershell
# Windows
Get-FileHash -Algorithm SHA256 .\bibliogon-launcher.exe
Get-Content .\bibliogon-launcher.exe.sha256
```

The hashes must match.

If your operating system warns about an unsigned binary, see the [Bibliogon installation overview](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/installation.md).

## What's new

### Picture-Book TipTap rich-text editing (4c-B-1)

The 3 layouts where text shape is unbounded — `image_top_text_bottom`, `image_left_text_right`, `text_only` — now mount a TipTap-backed `RichTextEditor` instead of a textarea. Authors can now apply Bold / Italic / Underline / heading levels / bullet + ordered lists / left-center-right alignment / font-family on per-page text. The 2 Tier-Property layouts (`speech_bubble`, `image_full_text_overlay`) keep textareas — their text shape is bubble/overlay-driven, not document-driven.

The Toolbar lives in the properties pane (D6-C hybrid placement) with 11 D1 MVP buttons plus the Font dropdown. Hidden for Tier-Property layouts.

Backward-compat: existing pages with plain-string `text_content` for a TipTap layout auto-wrap into a TipTap doc on first read — no data migration, no visual pop.

### 5-font OFL catalog + KDP-grade PDF embedding

Five OFL-licensed fonts shipped under `plugins/bibliogon-plugin-export/fonts/`:

| Font | License | Notes |
|------|---------|-------|
| Atkinson Hyperlegible | OFL 1.1 | Braille Institute. Default; designed for low-vision readability. |
| Andika | OFL 1.1 | SIL. Multi-script Latin coverage for international picture-books. |
| Comic Neue | OFL 1.1 | Modernised Comic Sans alternative. |
| Lexend | OFL 1.1 | Google + Thomas Jockin; reading-proficiency-research-backed. |
| OpenDyslexic | OFL 1.1 | Weighted-bottom letterforms for dyslexia-friendly reading. |

PDFs now embed the font files via `@font-face` with `src: url(file://...)` (was `src: local(...)` which was fragile in containers). KDP print requires embedded fonts; v0.35.0 PDFs meet that requirement.

The Font dropdown applies the chosen font to the ENTIRE page (picture-book convention: one page, one consistent font). Per-mark override filed as `PICTURE-BOOK-FONT-PER-MARK-OVERRIDE-01` for future.

### Picture-Book PDF export (Session 6)

WeasyPrint-backed 8.5×8.5 KDP-ready PDF generation:

- TipTap walker preserves bold / italic / underline / fontFamily marks + alignment + headings 1-3 + lists.
- PDF metadata embedded: Title / Author / Description / Generator / Lang.
- Export-PDF button in BOTH the PageEditor header AND the Book Metadata Design tab.

Other KDP formats (8×10, 8.5×11, landscape, bleed marks) filed as `PICTURE-BOOK-PDF-KDP-FORMATS-01` + `PICTURE-BOOK-PDF-BLEED-MARKS-01` for future.

### Speech-bubble polish

- **9-position anchor grid**: all 9 cells of the 3×3 grid clickable (was 5 — the 4 edge-midpoints are new).
- **Width slider**: 20-80%, default 40%. Replaces the single `size` knob.
- **Height slider**: 15-60%, default 30%. Brand-new dimension control.
- **Opacity slider**: 30-100%, default 100%. Unchanged.
- **Backend parity**: PDF export now honors all 9 anchor positions (was silently falling back to bottom-center for the 3 new ones).

### Visual region separator

The divider between image + text regions on `image_top_text_bottom` + `image_left_text_right` now uses the dedicated `var(--border-strong)` theme token + 2px width. Clearly visible across all 10 theme variants (5 palettes × light/dark); the previous opacity-mix approach was near-invisible on light themes.

### Medium-import async path

`POST /api/medium-import/import` is now async-job-backed with SSE-driven progress streaming. Long imports (200+ post archives with image downloads) no longer block on a synchronous spinner. The UI ships:

- Real progress bar with per-post events.
- Run-in-background dock badge: users can navigate away and check progress from any page.
- F5-recovery via context-backed state: refreshing during an import doesn't drop the connection.
- Go-to-comments link surfaces v0.31.0's parent-article matching + orphan detection.

### Release-automation pipeline

The release-automation audit shipped its four scopes:

1. **Aggregate Makefile targets**: `release-state` / `release-outdated` / `release-test` / `release-build` / `release-tag VERSION=` / `release-publish VERSION=`. Reduces manual-step surface.
2. **`sync-versions` also syncs `package-lock.json`**: closes the silent-version-drift gap discovered in v0.34.1.
3. **`make release-discover`**: open-set scan for hardcoded version literals outside `collect_targets()`.
4. **CI release-gate extension**: now runs `verify-plugin-locks` + `verify-docs-discipline` + launcher build smoke at tag-push time.

v0.35.0 is the first release cut entirely through this pipeline — first-test of the full automation chain. Reference: [`docs/development/release-automation.md`](https://github.com/astrapi69/bibliogon/blob/main/docs/development/release-automation.md).

### Other improvements

- **PageEditor ThemeToggle**: the picture-book editor header now exposes the same dark/light-mode toggle every other top-level surface has. Closed the 9th confirmed parallel-surface asymmetry.
- **CreateBookModal Authors-DB integration**: the modal uses the Authors-DB datalist + add-to-DB checkbox, matching the wizard pattern.
- **Tier-Property layout JSON leak fix**: textareas no longer show raw TipTap JSON when switched from a TipTap-rich layout. Defensive `extractPlainText` (frontend) + `_extract_plain_text` (backend) on the read path.

## Lessons-learned filed this cycle

- **Recurring-Component Unification Rule** (`.claude/rules/coding-standards.md`): UI patterns shared across 2+ surfaces extract NOW, not after the generic 3-duplicate threshold; UI duplication compounds visually + drives i18n + test churn.
- **Single-source-of-truth for cross-cutting concerns** (`.claude/rules/lessons-learned.md`): closed-set vs open-set discovery; the package-lock.json drift incident as the canonical example.
- **Half-wired-feature-lifecycle pattern**: extended with the 4th instance (the dead `kinderbuch.css` template, deleted in Session 6 Commit 1).
- **Smoke-findings = direct-action default** (memory): smoke findings get fixed in the current session unless the user explicitly defers; prior-session roadmap entries do not override the user's smoke-time direct request.
