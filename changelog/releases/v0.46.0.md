# Bibliogon v0.46.0 — 2026-06-05

A large release across four arcs: the **Dialog → Pages** migration on a new token-mapped **Tailwind v4 + shadcn/ui** foundation, **LAN Mode** (open Bibliogon from a phone on the same network), the **offline / local-first sync** stack (IndexedDB-backed offline editing with a background sync queue + conflict resolution), and **configurable default content/book types** plus a generic 9th content type.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

## Download

| Platform | File |
|----------|------|
| Windows | `bibliogon-launcher.exe` |
| macOS (Apple silicon) | `bibliogon-launcher-macos.zip` |
| Linux | `bibliogon-launcher-linux` (ELF binary) |

Each platform also ships a `*.sha256` checksum next to the binary.

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

## Action required

**None.** This release ships no schema migrations. Existing data, backups (`.bgb`) and projects (`.bgp`) are unaffected.

## What's new

### Dialog → Pages

Eight large dialogs are now deep-linkable full-page routes — each with a working Back button and a mobile-friendly layout, rendered through a shared `PageLayout` that carries the same app header (brand + theme toggle) as the rest of the app:

- `/books/new?type=<book_type>` (book creation)
- `/articles/new?type=<content_type>` (article creation)
- `/books/:bookId/export`
- `/writing-history` (global writing stats)
- `/books/:bookId/chapters/:chapterId/snapshots` (chapter version history)
- `/books/:bookId/git-backup` and `/books/:bookId/git-sync`
- `/help/shortcuts` (keyboard cheatsheet)

Small, context-bound dialogs (confirmations, transient-state wizards like the import / convert-to-book / AI-setup flows, and the onboarding nudges) stay dialogs by design — see `docs/architecture/dialog-to-pages-routes.md` for the full route map and classification.

### Deep-linkable chapters

The book editor's active chapter is now carried in the URL (`?chapter=<id>`). You can link directly to a chapter, and a snapshot restore returns you to the editor with the right chapter still selected.

### Foundation + performance

- **Tailwind v4 + shadcn/ui** — a token-mapped `@theme` bridge (Preflight omitted) so Tailwind color utilities resolve to the existing theme tokens; the theme system stays the single source of truth for color. A shadcn Dialog primitive now backs `AppDialog` and the confirmation dialogs.
- **`recharts` is lazy-loaded** with the writing-history page, moving ~330 kB out of the eager dashboard bundle.

### LAN Mode

Opt-in via `BIBLIOGON_LAN_MODE`: Bibliogon serves the built frontend and the API from a single port so a phone or tablet on the same network can reach the desktop instance. A startup QR banner shows the auto-detected LAN URL, a PIN gate protects access (session cookie + lockout), and a Settings > About card shows the URL + PIN + QR. New `make dev-lan` / `make build-frontend` targets drive it.

### Offline mode + background sync (mobile / local-first)

An `IStorageService` seam with two backends — `ApiStorage` (online) and `DexieStorage` (IndexedDB, offline) — selected by a connectivity monitor. "Take offline" selectively downloads a book for offline editing; offline writes queue and a background sync engine replays them on reconnect, with conflict detection + resolution and status toasts. Desktop is unaffected: the whole stack is opt-in and dynamically imported so it stays out of the desktop bundle.

### Defaults + content types

- **Configurable default book-type and content-type** (Settings > Verhalten): the dashboards' "new" actions and the create pages honor your chosen default, and the SplitButton primary label reflects it.
- **Generic "Article" content type** — a 9th, unspecified content type (no per-type metadata, like Essay) alongside the existing 8.

### Fixes

- **Blogpost (the default) is no longer hidden** from the New-Article type menu; a stale PWA Service Worker that masked the fix is now disabled in dev and self-deregisters.
- Dialog → Pages dark-mode + centering/logo fixes.
