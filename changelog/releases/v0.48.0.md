# Bibliogon v0.48.0 — 2026-06-09

The **Maximal Offline** release — the largest in the project's history (111 commits since v0.47.0). The static GitHub Pages PWA now reaches **full desktop feature parity**: it boots with no backend and does *everything* the desktop does through a seeded IndexedDB and the `IStorageService` seam — prose, articles, picture-books, comics, the Story Bible, the Storyboard, authors, chapter-labels, publications and article-platforms all work offline, exports run in the browser across six formats, AI runs directly against the user's own provider key, and Medium archives import client-side. A single `guardedFetch()` choke point keeps the backendless build at **zero `/api` requests**, enforced by an E2E hard gate. Backend-only features (Pandoc/LaTeX export, Git sync, audiobook TTS, LAN mode) are the only gated exceptions, shown with a translated "requires the desktop app" hint rather than a broken button.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

You do not need the desktop app to try Bibliogon: the offline web app at [astrapi69.github.io/bibliogon/](https://astrapi69.github.io/bibliogon/) runs entirely in your browser, with no install and no backend.

## What's new

### Added

- **Maximal Offline — full desktop parity in the browser.** The static GitHub Pages PWA does everything the desktop does. The Story Bible, Storyboard, picture-book and comic editors, authors, chapter-labels, publications and article-platforms now all read and write through the `IStorageService` seam, so they work entirely offline against a seeded IndexedDB. The seam was extended to cover every data entity; only four genuinely browser-impossible features remain backend-gated (see Changed).
- **Client-side export engine.** Six formats render entirely in the browser with no Pandoc: Markdown, HTML, plain text, PDF, EPUB and DOCX. Offline, the dedicated export surface uses the client engine and produces real downloads.
- **Export-engine chooser.** Settings > Export lets you pick `auto` / `client` / `backend`: `auto` uses the client engine offline and the backend (Pandoc/LaTeX) when available, or you can force either.
- **AI offline.** `AiGenerateButton` and the full `.biblio.yaml` template-fill flow run directly from the browser against the user's own provider key (stored in IndexedDB, sent only to the provider, never to `/api`).
- **Client-side Medium import.** Medium HTML-export archives are parsed and imported in the browser via `DOMParser` + `fflate` ZIP handling — no backend round-trip.
- **Seed pipeline.** `make generate-seed-data` reads the canonical backend YAML single-source-of-truth and emits 12 committed JSON files under `frontend/src/storage/seed/` (i18n catalogs, settings defaults, book/content types, plugin metadata) whose shapes mirror the `/api` responses; `DexieStorage` populates its reference tables on first init via an idempotent `ensureSeeded()`.
- **`guardedFetch()` single choke point.** Every client network call flows through one egress that rejects before any fetch on the backendless build, guaranteeing zero `/api` requests in Dexie mode (with an E2E hard gate as a regression backstop).
- **Settings > About.** Shows the app version, build hash and build date.
- **Service-worker auto-update.** The PWA checks for a new build on focus, on visibility change, and hourly, so users pick up deploys without a manual hard-reload.
- **GitHub Pages deploy infrastructure.** Split deploy (app on `astrapi69.github.io/bibliogon/`, docs on `astrapi69.github.io/bibliogondocs/`) with a `404.html` SPA redirect for deep links.
- **`@astrapi69/entity-kit` integration (PoC).** The book TrashView is the first surface migrated to the shared entity-kit views (descriptor-driven trash + tile views).
- **Responsive mobile.** A mobile audit plus fixes: collapsible sidebars and 44px minimum touch targets across interactive controls, for the phone/tablet surfaces the PWA now serves.
- **Repository-pattern migration.** 11 routers/entities moved to the repository interface seam (service ↔ data boundary).
- **Help docs.** New Web-App-usage, Browser-Export and Story-Bible-offline pages (DE + EN); ROADMAP updated to the current state.

### Fixed

- **DexieStorage lazy-load race.** A first-render race left the book/content-type dropdowns empty and made the book cover fail to display on a cold load; DexieStorage is now preloaded before the first render when an explicit Dexie mode is set, and `useAssetUrl`/cover resolution no longer misses on a cold mount.
- **Settings forms clobber.** A parent-gate plus per-form guards stop concurrent settings loads from overwriting each other.
- **Offline connectivity monitor.** Correctly starts mid-session so the app reacts to going offline/online after load.
- **Article `.split` crash.** Hardened against an undefined i18n key (`t()` fallback).
- **Menu single-line header.** Switches on a fixed Tailwind breakpoint (1200px) instead of toggling on language / default-type change.
- **Settings "skip non-destructive confirmations" persistence.** The toggle now persists (the update schema gained a `behavior` field).
- **View-mode toggle-during-load revert.** A view toggle made while the persisted preference is still loading is no longer clobbered back by the late mount-load.
- **Picture-book / comic add-page-during-load clobber.** The add-page button is gated until the mount `pages.list` resolves, so a page added during load can't be wiped by the late list-resolve.
- **Article grid-to-list flash.** The article list defaulted to `list` but rendered `grid` first then switched on mount, detaching the bulk-action-bar / checkboxes mid-interaction; the list now defers rendering until the view mode resolves.
- **Convert-to-book wizard listbox race.** The sort-strategy dropdown is reliably closed before advancing a step.
- **Seed privacy.** The committed, publicly-served seed JSON no longer carries any personal data (author name + pen names blanked, like the AI key).
- **Offline guard toasts suppressed.** Offline-rejected `/api` calls log a `console.warn` instead of surfacing a red error toast.
- **Credentials chain.** The secret fallback chain (`app.yaml` < user override < env-var) and `app.paths` resolution work correctly out of the box.
- **Offline-PWA E2E spec.** Fixed an ESM/CJS load error in the offline spec.

### Changed

- **DEXIE-MODE-REGEL → Maximal Offline.** The architecture rule now mandates that the web app do everything via Dexie; the offline gate is the exception, reserved for the four genuinely browser-impossible features (Pandoc/LaTeX export, Git sync/backup, audiobook TTS, LAN mode). (`architecture.md`)
- **CSS-first → Tailwind-first.** New visual/layout work goes through Tailwind utilities; `global.css` is frozen (existing classes reused, never extended). (`coding-standards.md`)
- **No inline comments.** Code is self-explanatory; explanations live in docstrings / TSDoc. (`code-hygiene.md`)
