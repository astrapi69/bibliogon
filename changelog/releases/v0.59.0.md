# Bibliogon v0.59.0 — 2026-06-28

The **offline-authoring-depth + writing-insights + God-file burn-down**
release. The editor gains browser-native read-aloud and offline AI
grammar-check + translation, a new Writing-Statistics dashboard turns your
`writingSessions` into progress rings, charts and a streak heatmap, book
templates work fully client-side in offline mode, and a searchable
keyboard-shortcuts overview lands. Under the hood the WARN-tier God-file
burn-down decomposes ten more modules under 500 lines. No schema migrations;
existing data + `.bgb`/`.bgp` are unaffected.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher manages Docker for a local copy of the Bibliogon repository: it builds the Docker images from your copy and opens the app in your browser. You need the Bibliogon source on disk first (a `git clone`, or the source ZIP unpacked — default `~/bibliogon`); the launcher does not download Bibliogon itself. The first build takes 5-10 minutes (~2 GB disk space).

You do not need the desktop app to try Bibliogon: the offline web app at [astrapi69.github.io/bibliogon/](https://astrapi69.github.io/bibliogon/) runs entirely in your browser, with no install and no backend.

## Download

| Platform | File |
|----------|------|
| Windows | `bibliogon-launcher.exe` |
| macOS (Apple silicon) | `bibliogon-launcher-macos.zip` |
| Linux | `bibliogon-launcher-linux` (ELF binary) |

Each platform also ships a `*.sha256` checksum next to the binary.

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

## What's new

### Added
- **Writing-Statistics dashboard (`/statistics`).** A today progress ring with a yesterday comparison, a last-7-days chart with the daily average as a reference line, per-book project progress with an estimated completion at the recent pace, and a contribution-style streak heatmap — aggregated client-side so it works online and offline. (#668)
- **Keyboard-shortcuts overview dialog** — searchable and context-aware across every editor and app shortcut. (#667)
- **Browser-native read-aloud (Text-to-Speech)** in the editor via the Web Speech API — capability-detected and running fully offline with no backend, distinct from the desktop-only audiobook export. (#666)
- **Offline AI grammar-check + translation** in the editor — run browser-direct against the configured AI provider, gated disabled-until-a-key-is-set (`ai-grammar` / `ai-translate`). The server LanguageTool + DeepL/LMStudio paths stay desktop-only. (#661, #669)
- **Client-side offline book templates** — "Aus Vorlage" works fully client-side in Dexie mode. (#670)
- **Settings — preview/test-version banner, build-info + share-app QR section.** (#644)
- **GH-Pages static preview deploy** (#639) plus a static-smoke gate that exercises the backendless build. (#640, #641)

### Changed
- **God-file WARN-tier burn-down** — decomposed under 500 lines: ArticleEditor via sidebar extraction (#207/#634), GitSyncPage (#648), ConvertToBookWizard (#650), LayoutConfigImageRow (#652), useComicBookEditor (#654), AboutSettings (#676), CreateBookForm (#678), the `platform.ts` API client (#680), ComicBubble (#682), and CommentsAdminSection (#684). `Editor.tsx` is whitelisted as the irreducible TipTap editor shell. (#656)
- **Ratchet baseline convention** — directory/file-size baselines renamed to the extension-less `.dirsize-` / `.filesize-` root convention with headers. (#637, #646)

### Fixed
