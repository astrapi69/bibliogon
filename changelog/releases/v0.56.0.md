# Bibliogon v0.56.0 — 2026-06-18

The **first published release since v0.51.0.** The v0.52.0 → v0.55.0 tags were
cut but never published as GitHub releases, so this release brings the published
stream back in sync: it carries the whole v0.52.0 → v0.55.0 arc plus this
cycle's polish. The headline fix this cycle: **bulk-delete no longer silently
fails above 200 selected items.** No schema migrations; existing data +
`.bgb`/`.bgp` are unaffected across the entire span.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

You do not need the desktop app to try Bibliogon: the offline web app at [astrapi69.github.io/bibliogon/](https://astrapi69.github.io/bibliogon/) runs entirely in your browser, with no install and no backend.

| Platform | File |

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

## What's new

### New in v0.56.0

#### Added
- **Clear-data preview** — Settings previews the event-log and image-cache contents before you clear them, so a destructive clear is an informed choice. (#408)

#### Changed
- **Downloads** keep the native `downloadBlob` path (file-saver dropped); the helper is hardened and de-duplicated across the export surfaces. (#416)
- **Backend** — the ASCII filename-slug logic is consolidated into a single shared helper. (#390)
- **Visual-regression infra** — Phase-1 multi-viewport screenshot baselines and a nightly CI job. (#411)

#### Fixed
- **Bulk-delete above 200 selected items** — selecting more than 200 articles or books and choosing Löschen did nothing (no toast, no trash). The frontend delete handlers had reused the export cost-profile cap (200) as a guard; delete is uncapped (a DB UPDATE/DELETE per row), so the cap is removed for both Articles and Books. Reproduced with 1387 selected articles. (#417)
- **Quality-report chapter order** — report chapters are ordered by the sidebar matter groups (front / main / back) rather than raw position. (#415)
- **Custom combobox** — a pending typed value is committed when the dropdown is closed by clicking outside, instead of being discarded. (#404)
- **Mobile editor sidebar** — closes via a click-to-close overlay across all four overlay-sidebar editors. (#405, #406)

- **v0.52.0** — the **offline-depth + stability-hardening** release: the offline import wizard restores `.bgb` full-data backups client-side; the Medium import moves to a dedicated page with per-post progress + thumbnails; author pen names become a proper `<select>`; the storage seam is hardened (nine direct-seam bypasses removed); LaTeX export bug fixes.
