## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

## Download

| Platform | File |
|----------|------|
| Windows | `bibliogon-launcher.exe` |
| macOS (Apple silicon) | `bibliogon-launcher-macos.zip` |
| Linux | `bibliogon-launcher-linux` (ELF binary) |

Each platform also ships a `*.sha256` checksum next to the binary.

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

```powershell
# Windows
Get-FileHash -Algorithm SHA256 .\bibliogon-launcher.exe
Get-Content .\bibliogon-launcher.exe.sha256
```

The hashes must match.

If your operating system warns about an unsigned binary, see the [Bibliogon installation overview](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/installation.md).

## What's new

### Storyboard View for Picture Books

A drag-reorder grid view for picture-book pages with per-page annotations, replacing the older single-list page management for storyboard-style overview. 16 commits across 2 sessions.

- **Drag-reorder grid:** rearrange pages visually; saves via the existing `/api/books/{id}/pages/reorder` endpoint with stale-client detection.
- **Per-page notes:** inline notes textarea on each card (auto-save on blur).
- **Story-beat tag:** select from a 6-value enum (Exposition / Inciting Incident / Rising Action / Climax / Falling Action / Resolution) for at-a-glance dramatic structure.
- **Mood-color palette:** 10 preset swatches (Sunny / Passionate / Calm / Dreamy / Peaceful / Adventurous / Tender / Somber / Mysterious / Gentle) without an external color-picker dependency.
- **Act-group label:** free-text inline editing for act / chapter boundaries; visually groups consecutive same-act cards.
- **Click-to-navigate:** click any card to jump to that page in the regular page editor.
- **Page-routing:** Storyboard mounts via `?view=storyboard` query-param flip from a Storyboard button in the picture-book editor header.

Backlog item closed: `PICTURE-BOOK-STORYBOARD-VIEW-01`. Three P5 follow-up items filed for future expansion: `PICTURE-BOOK-STORYBOARD-OPERATIONS-01` (add-page-in-between / duplicate / split / merge / print), `STORYBOARD-MOOD-FREE-PICKER-01` (free-hex color picker), `STORYBOARD-DRAG-CROSS-GROUP-ACT-UPDATE-01` (auto-update act_group on cross-group drag).

### Picture-Book Text-Stack — Fix B per-layout namespace + Tier 1+2 across all image layouts

A coherent multi-session text-configuration arc that fixes a long-standing layout-config bleed between picture-book layouts and adds the Tier 1 + Tier 2 properties (8 Visual-Style + 6 Typography fields) to every image-based layout. 18 commits across 2 sessions.

- **Fix B per-layout namespace** (supersedes Fix A from v0.33.1): `Page.layout_config` now nests configurations under each layout's name (e.g. `{speech_bubble: {...}, image_top_text_bottom: {...}}`). Switching speech_bubble → image_full_text_overlay → speech_bubble preserves the original speech_bubble configuration; no more null-purge on switch. Legacy flat configs auto-migrate to the namespaced shape on the next write.
- **Tier 1 (Visual Style) + Tier 2 (Typography) sections** added to `image_top_text_bottom`, `image_left_text_right`, and `image_full_text_overlay` layouts. Each section ships 14 fields total (8 visual + 6 typography) plus the established `speech_bubble` precedent.
- **Overlay-specific width + height sliders** (`image_full_text_overlay`): narrow the text band horizontally and limit vertical height, working together with the existing text-position + opacity controls.
- **Shared style helper:** `computeTierTextStyles` (TypeScript) + `_compute_tier_text_style` (Python mirror) — RCU 3-surface extraction so the editor preview and the WeasyPrint PDF walker render the same Tier configuration.

Backlog items closed: `PICTURE-BOOK-PAGE-TEXT-TIPTAP-INTEGRATION-01` (D2-D6 confirmed shipped during pre-inspection), `PICTURE-BOOK-OVERLAY-TEXT-TIER-PROPERTIES-01`, `PICTURE-BOOK-TEXT-CONFIGURATION-01`.

### Active text-conversion on layout-switch

Companion data-hygiene win to the Text-Stack work. When a page's layout switches FROM a TipTap layout (`image_top_text_bottom`, `image_left_text_right`, `text_only`) TO a Tier-Property layout (`speech_bubble`, `image_full_text_overlay`), the PATCH now carries the extracted plain-text version of `text_content` alongside the layout flip. The DB shape is cleaned at switch time; subsequent reads don't pay the parse cost; rows no longer carry a stringified TipTap doc in a Tier-Property layout. Symmetric direction (Tier-Property → TipTap) leaves `text_content` untouched because `parseTextContentToJson` already wraps plain text on read.

Backlog item closed: `PICTURE-BOOK-LAYOUT-SWITCH-TEXT-CONVERSION-01` (was P3, trigger-gated; shipped against the self-actionable trigger as part of the v0.39.0 release-prep arc).

## Compatibility

- **Schema migration**: 4 new nullable columns added to `Page` (`notes`, `story_beat`, `mood_color`, `act_group`). Alembic migration auto-applies on first start; existing rows get `NULL` for all four fields. No data loss.
- **`layout_config` migration**: legacy flat configurations auto-migrate to the namespaced shape on the next write per the Fix B path. Reads handle both shapes via `readLayoutNamespace` / `read_layout_namespace`. Configs written under v0.34.0+ Fix A's purge-on-switch behavior are still gone (Fix A erased them); from this release forward configurations are preserved.
- **`text_content` shape**: post-switch rows now carry plain text in Tier-Property layouts (no more raw JSON strings). Existing dirty rows in the wild are handled by the defensive `extractPlainText` read at display time.
- **All existing `?view=...` URLs preserved**: Storyboard is a new `?view=storyboard` variant, not a replacement for the default page editor.
- **i18n**: ~120 new keys across the Storyboard + Text-Stack + sidebar + pagination + display-settings surfaces × 8 catalogs.
