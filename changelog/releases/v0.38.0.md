# Bibliogon v0.38.0 — 2026-05-26

A Settings-UX overhaul release. 30 commits since v0.37.0 (one day after the v0.37.0 ship). The audit filed at v0.37.0 release-time (8 follow-up items in `056fdfd`) drove four coordinated work streams that, together, restructure the Settings page from a 13-tab horizontal scroller into a left-sidebar grouped nav with consistent toggles + per-section descriptions + dedicated tabs.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

## Download

| Platform | File |
|----------|------|
| Windows | `bibliogon-launcher.exe` |
| macOS (Apple silicon) | `bibliogon-launcher-macos.zip` |
| Linux | `bibliogon-launcher-linux` (ELF binary) |

Each platform also ships a `*.sha256` checksum next to the binary.

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

```powershell
# Windows
Get-FileHash -Algorithm SHA256 .\bibliogon-launcher.exe
Get-Content .\bibliogon-launcher.exe.sha256
```

The hashes must match.

If your operating system warns about an unsigned binary, see the [Bibliogon installation overview](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/installation.md).

## Action required

**None for existing users.** v0.38.0 is purely a Settings-page UX overhaul (sidebar redesign, tab consolidation, Toggle migration) + a dashboard-nav layout fix. No schema changes; no migrations; existing `?tab=` deep-link URLs preserved including legacy `?tab=author` and `?tab=authors_database` redirects.

## What's new

### Settings page: horizontal tabs → left sidebar nav

The 13-tab horizontal bar at 900px max-width produced a horizontal scrollbar once the Autor + Autoren-Datenbank consolidation closed earlier in the day (user-pull trigger fired same-session). The new layout is a two-column grid (220px sidebar + 900px content inside a 1180px outer container) with 5 grouped sections:

- **Darstellung** — Erscheinungsbild, Verhalten, Editor
- **Inhalt** — KI-Assistent, Autoren, Themen
- **System** — Plugins, Kommentare, Backups, Erweitert
- **Info** — Über, Unterstützen (when donations are configured)
- **Gefahrenzone** — visually separated with a red accent + divider above

The mobile dropdown (<=768px) keeps the previous hamburger pattern, now with separators between groups so the structure surfaces on mobile too. All 13 existing `settings-tab-{value}` testids preserved — every URL-based deep-link and bookmark keeps working.

### Settings Phase 1 — 7 quick wins (SETT-QW-1..7)

Tightens the existing Settings page without restructuring tabs:

- **Dashboard-view controls** grouped into one sub-card so the 5 related selectors don't look like 5 unrelated rows
- **SSH-Key block** migrated into its own card with a header
- **Editor tab** extracted from Allgemein
- **`sectionTitle` CSS class** standardized across every settings panel
- **`HelpText` component** extracted from the 20-odd ad-hoc inline `<small>` helpers
- **White-Label section** turned into a collapsible (default closed; rarely-used by single-user installs)
- **`SectionHeader` composition component** + per-section descriptions — every settings tab now opens with a one-paragraph "what this tab does" cue

### Settings Phase 2 — Allgemein split (SETT-PHASE-2)

The catch-all "Allgemein" tab decomposed into three focused tabs:

- **Erscheinungsbild** — appearance (themes, dashboard views, per-page selectors)
- **Verhalten** — behaviour (autosave, default status, trash auto-delete, default language)
- **Erweitert** — advanced (debug, white-label collapsible)

Obsolete `AppSettings.tsx` removed. Every setting kept its existing semantics; only the tab grouping changed.

### Settings Phase 3 — Toggle composition (SETT-PHASE-3)

New `Toggle` composition component for the canonical checkbox + label + HelpText pattern. 5 canonical-shape sites migrated (VerhaltenSettings 3 + AI 1 + AudiobookSettingsPanel `readChapterNumber` 1). The remaining 6 checkbox sites in Settings sub-components are intentional design-intent exemptions documented in `Toggle.tsx`'s docstring (per the design-intent-axis lessons-learned rule): 2 inline label-after-checkbox sites where Toggle adds no value, 3 list-row sites with a different structural shape, and 1 generic plugin-setting renderer that's out of scope for a Settings-specific component.

### Settings — Autor + Autoren-Datenbank consolidated

The two formerly-separate "Autor" (your profile) and "Autoren-Datenbank" (the known-authors registry) tabs merged into a single **Autoren** tab. The sub-components stay independent; only the tab wrapper changed. Legacy `?tab=author` and `?tab=authors_database` URLs redirect to `?tab=autoren`.

### Article Dashboard top-nav alignment

The top navigation menu was jumping horizontally when switching between Book Dashboard and Article Dashboard — the standalone "Aus Medium importieren" button on Articles widened the toolbar by ~150px relative to Books, shifting every middle button leftward. Fix: the Medium-import button collapsed into a chevron disclosure on the existing Importieren button (split-button pattern mirroring `newBookGroup` on Dashboard.tsx); added a second `headerSeparator` to mirror the Books-side structure. Article width before: ~1014px; after: ~908px (matches Book Dashboard).
