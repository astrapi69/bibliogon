# Bibliogon v0.44.0 — 2026-06-01

The Scrivener-parity + visual-planning release. Two coordinated Scrivener-gap clusters (ergonomics + drafting) plus Bibliogon's standout new visual feature — an interactive Story Bible relationship graph.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser. The first launch takes 5-10 minutes (Docker images build, ~2 GB disk space).

## Download

| Platform | File |
|----------|------|
| Windows | `bibliogon-launcher.exe` |
| macOS (Apple silicon) | `bibliogon-launcher-macos.zip` |
| Linux | `bibliogon-launcher-linux` (ELF binary) |

Each platform also ships a `*.sha256` checksum next to the binary.

## Verifying downloads

```bash
# macOS / Linux
shasum -a 256 bibliogon-launcher-<platform>
cat bibliogon-launcher-<platform>.sha256
```

## Action required

**None.** The new schema (`chapter_labels` table, `chapters.status` /
`label_id`, writing-goal + per-day `writing_sessions` columns,
`chapter_versions.name` / `is_manual`, `books.graph_layout` and writing
targets) is applied automatically on first start (the FastAPI lifespan
runs the Alembic migrations). This release also fixes the Alembic
upgrade chain, which previously crashed at the chapter-status-labels
migration on incremental upgrades. Existing data, backups (`.bgb`) and
projects (`.bgp`) are unaffected.

## What's new

### Scrivener ergonomics
- **Composition mode** — distraction-free writing (Ctrl+Shift+D / Esc): hides app chrome, paper backdrop, paragraph dimming, typewriter scrolling.
- **Chapter status + labels** — per-chapter workflow status plus a user-definable, colour-coded label set per book; chips on the Storyboard + Outliner.
- **Writing goals + streak** — per-book word target + deadline with an auto-computed daily target, per-chapter targets in the DB, and a Dashboard daily-goal widget with a streak counter.
- **Chapter Outliner** — a sortable, inline-editable spreadsheet of the chapters (`?view=outline`).

### Drafting + history
- **Chapter snapshots** — named manual snapshots before major edits, a line-diff view against the current content, and one-click restore (which snapshots the current state first).
- **Writing history** — daily word-count bar chart, summary stats (total / active days / average / current + longest streak), per-book + per-chapter breakdown, and CSV export.
- **Word/EPUB import** — documentation for the Pandoc-based DOCX/EPUB import in the import wizard.

### Story Bible relationship graph
- An interactive node-graph (`?view=relationships`): entities as shaped, colour-coded nodes; relationships as colour-coded edges.
- Drag from one node to another to create a relationship (type + note); click an edge to delete it.
- A node detail panel ("Open in editor" / "Show appearances") and double-click-to-open.
- Per-book layout persistence, a Reset-layout button, and PNG export for marketing screenshots or the Story Bible document.

### Quality
- Fixed a pre-existing Alembic upgrade-chain crash (batch FK-add table recreate) and added a regression gate that runs the real `alembic upgrade head` against a throwaway database.
- New dependencies (all MIT, 0 high/critical advisories): `recharts`, `@xyflow/react` v12, `html-to-image`.
