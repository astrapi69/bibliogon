# Bibliogon v0.26.5 — 2026-05-04

Hotfix for v0.26.4. v0.26.4 attached Linux and macOS launcher binaries; Windows release-event run still failed at the install.sh sync check. v0.26.5 fixes the Windows path; this is the version Windows users should install. Linux and macOS users on v0.26.4 carry the same binaries.

## What broke on Windows in v0.26.4

GitHub Actions Windows runners default to `core.autocrlf=true`. On checkout, text files (including `install.sh`) gain CRLF line endings. The Phase 2 tooling regenerates `install.sh` from `install.sh.template` via `bash` (always LF output) and compares to the committed file via `diff -q`. CRLF vs LF mismatch failed the check, even though the file content was identical.

## Fix

New `.gitattributes` at repo root pins shell scripts and the install template to `text eol=lf`:

```
*.sh         text eol=lf
*.template   text eol=lf
install.sh   text eol=lf
```

Git on every platform — regardless of `autocrlf` — checks these files out as LF. Linux and macOS were already LF; Windows now matches.
