# Bibliogon v0.31.0 — 2026-05-13

The "deep features" release: universal AI templates (per-field-class selection, three workflow modes including an external-AI YAML round-trip), bulk-delete (article + book, no-cap with a destructive type-to-confirm gate + soft-delete Undo toast), the Medium HTML archive importer (10th plugin) and its comments routing, and a sweep of theme tokens + i18n review-status hardening. 157 commits since v0.30.0.

## Before you install

Bibliogon runs in Docker. You need Docker Desktop installed and running before starting the launcher.

- [Docker installation guide (English)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/en/install/docker-desktop.md) — includes a "Is Docker safe to install?" section
- [Docker-Installationsanleitung (Deutsch)](https://github.com/astrapi69/bibliogon/blob/main/docs/help/de/install/docker-desktop.md) — mit Abschnitt "Ist Docker sicher zu installieren?"

The launcher detects Docker, downloads Bibliogon automatically, and opens it in your browser.

## Action required

**If you developed against Bibliogon outside Docker (i.e. with `make dev`) on v0.30.x or earlier:** your `backend/config/backup_history.json` and any user-installed plugin under `backend/plugins/installed/` auto-migrate to the canonical data dir on first boot of v0.31.0. A `.migrated-2026-05-13` breadcrumb is written next to the old path so the move is verifiable; the breadcrumb file is safe to delete after a successful first boot. Production Docker installs and standard launcher installs are unaffected (production already writes to the named `/app/data` volume).

## What's new

**Universal AI templates for Articles and Books.** A self-explanatory YAML template format that ships with three first-class workflows: (A) the built-in AI provider, (B) a custom local endpoint (LM Studio / Ollama, configured via a new "Custom" preset in Settings → AI), and (C) external AI via YAML round-trip. The third mode is the load-bearing one — export the template from an article or a book, paste the YAML into Claude.ai / ChatGPT / any LLM playground (the file carries its own fill-rules + style-rules at the top so the AI has all the context it needs), upload the filled YAML back. The same `.biblio.yaml` artifact powers all three modes; no out-of-band documentation needed. Per-field-class selection (cover prompt / SEO / tags / excerpt / description / chapter summaries / inline image prompts) via a new `FieldClassDialog`. Bulk operations with a configurable per-batch cap (default 50, raise via `ai.bulk.max_ai_fill` / `ai.bulk.max_ai_template` in `backend/config/app.yaml`). Bulk AI-fill shows a live cost projection during a running job (`~$total projected` badge on the dock, per-item average pill in the expanded modal, hidden until the first priced response lands, replaced by the authoritative total on completion).

**Bulk-delete for articles and books.** Per-tile checkboxes + filter-aware select-all + a delete-dropdown on the bulk-action bar (`In Papierkorb verschieben` for soft-delete with an Undo toast that restores every row, `Endgültig löschen` for permanent delete behind a new `TypeToConfirmDialog`). The destructive-confirm dialog uses a numeric type-to-confirm gate: type the exact count of selected items to enable the confirm button. The gate is paired with a filter-description line so the deleted scope is visible. The single-item delete flow stays on the per-card menu — the bulk-delete dropdown stays disabled at count < 2. The 200-row cap that was copied from `bulk-export` was removed for delete (SQL bulk DELETE is sub-second on thousands of rows; the cap was UX-hostile when a user selected 209 imported Medium articles). i18n × 8 catalogs.

**Medium HTML archive importer (10th plugin).** Imports a Medium HTML export ZIP (Settings → "Download your information") and produces Article + Publication + provenance rows per `posts/*.html`. Image references download to local storage by default per the data-sovereignty design decision. Re-imports of the same archive deduplicate against `Article.canonical_url`. Auto language detection (Medium's export carries no `lang` attribute; statistical detection via langdetect). First body image becomes the article's `featured_image_url` (off-by-toggle). Dedicated `/articles/import/medium` page route chosen over a modal because the typical Medium archive runs minutes — the long processing time + the structured results table justify a top-level surface.

**Medium comments routing.** Medium's HTML export treats user-written responses (short reply-shaped notes to other articles) as standalone HTML files indistinguishable from articles at the file level. The walker now runs a length + structure heuristic and routes detected comments to a new `article_comments` table instead of polluting the article dashboard. Three import modes (`as_comments` default, `as_articles` legacy, `skip`). Read-only `ArticleCommentsPanel` in the editor sidebar; comment-count badge on `ArticleCard` and the article list view; Settings → "comments" tab with source filter + paginated table + per-row delete.

**Theme + i18n hardening.** Four new CSS variables defined across all 6 palette / dark-mode combinations (`--surface-2`, `--danger-bg`, `--success`, `--warning`) — 9 new components had been referencing these tokens with hex fallbacks, so every consumer was silently falling through to its light-mode hex value under the Cool Modern / Nord / Classic / Studio / Notebook palettes. Backend i18n catalogs for es / fr / el / pt / tr / ja now carry an explicit `_meta.review_status` marker for the three v0.31.0 namespaces (`ai_template`, `bulk_ai_fill`, `comments`) which ship passthru-English pending native-speaker review (follow-up tracked as `I18N-NATIVE-REVIEW-V031-01`). Companion docs file at `backend/config/i18n/REVIEW_STATUS.md` explains the marker and the PR-based correction workflow.

**A1 bulk-delete E2E smoke spec.** Closes the critical gap identified by the v0.31.0 pre-release coverage audit. Three tests pinning the data-destructive guards: count=1 dropdown disabled, soft-delete + Undo restore, permanent delete type-to-confirm gate (disabled-when-empty → disabled+error on wrong count → enabled on correct count → all rows gone with empty trash list confirming permanent path skips trash entirely).

## Internal

- **Path-isolation defense-in-depth.** `backend/app/backup_history.py` and `backend/app/routers/plugin_install.py` now resolve their write paths via `app.paths.get_data_dir()` instead of CWD-relative literals / `BASE_DIR`-relative constructs. The previous code crashed in dev-docker because the bind-mounted project tree inherits the host's UID, not the container's `bibliogon` user. Production Docker (named volume + `chown -R bibliogon`) was never affected. The broader pattern (10+ similar writes in `settings.py`) is filed as `PROD-WRITES-ARCHITECTURE-01` (P3) and deliberately deferred — the production deployment is correct; only the dev environment needs the rest of the cleanup.

- **Drift gates.** Two CI release-gate drifts caught by the pre-release verification: a hardcoded `__version__ = "1.0.0"` literal in the new medium-import plugin (replaced with `importlib.metadata.version(...)` to match `plugin-git-sync`), and an out-of-sync `mkdocs.yml` nav for the AI Templates help section (regenerated via `make sync-mkdocs-nav`). Both would have rejected the v0.31.0 tag push.

- **Documented limitations.** Medium's HTML export strips every SEO meta tag (`<meta name="description">`, `og:*`), every tag entry, and every `lang` attribute; the export-stripping pattern generalises beyond Medium and is now a lessons-learned entry. Comments that other people wrote on the user's articles are NOT in the export (Medium's export is "your data only", consistent with Twitter / Reddit / Discord); manual entry is the workaround and the schema is prepared (`ArticleComment.imported_from = "manual"` is reserved for a future P5 workflow).

- **Pre-release audits.** Three audit reports cover the v0.31.0 surface:
  - `docs/audits/pre-release-verification-2026-05-12.md` — every test layer, every static check, every drift gate.
  - `docs/audits/pre-release-coverage-2026-05-12.md` — gap audit by new feature, critical gaps surfaced as priority A.
  - `docs/audits/pre-release-ux-2026-05-12.md` — i18n parity matrix, accessibility primitives, theming compliance, cross-component pattern check.

- **New P3 backlog items filed in this cycle**: `I18N-NATIVE-REVIEW-V031-01`, `PROD-WRITES-ARCHITECTURE-01`, `BACKUP-HISTORY-SINGLETON-01`.

## Verifying downloads

Each release artifact is published with a SHA-256 hash. Verify before installing on production machines:

```bash
sha256sum <downloaded-file>
```

The expected hash is listed alongside each artifact on the GitHub release page. The `install.sh` one-liner verifies the launcher binary automatically.
