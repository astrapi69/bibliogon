# Bibliogon v0.36.0 — 2026-05-23

Largest release since v0.30.0 by commit count. Centers on three strategic streams that all opened during the v0.35.x cycle and matured together:

1. **plugin-comics** matured from scaffold (v0.35.1 v1.0.0) to a working multi-panel + multi-bubble comic-book editor (v1.1.0) — full PDF dispatch via the new `export_execute` hookspec wiring.
2. **KDP Publishing Wizard** shipped Phase 1 (3-step) and Phase 2 (5-step) — XState v5 state machine, server-side persistence, conflict-resolution banner. Largest greenfield frontend feature of the cycle.
3. **PluginForge v0.7.0 → v0.10.0 adoption** brought the 3-source plugin metadata pattern, DiscoveryResult severity filtering, version-gating, `/api/admin/rediscover`, and the `target_application` / `app_id` identity declaration into Bibliogon.

Plus 17 other coherent surfaces — Book-Types SSoT, WizardShell + 3-site migration, browser-native fullscreen across 3 editors, Backups consolidation into Settings, the 9th About-dialog Settings tab, picture-book PDF format dropdown + bleed-marks, speech-bubble Tier 1/2 properties, Author input migrated to Pattern A across 4 surfaces, the BulkActionBar 3-site RCU extraction, page-delete UI, browser-native word-wrap toggle, Medium-import excerpt auto-fill, multi-book-type onboarding, KDP-categories autocomplete wired to the 26-entry catalog, user-overlay plugin-enable migration, BookPublishingState + ArcReviewer schema + CRUD, and the I18N-articles namespace cleanup.

230 commits since v0.35.1. +59765 / -2909 lines across 331 files.

## What's new

### plugin-comics v1.1.0 — multi-panel + multi-bubble comic editor

Discriminator: `Book.book_type = "comic_book"`. Scaffolded in v0.35.1 as a placeholder; this release delivers the full editor.

- **Backend**: new `comic_panels` + `comic_bubbles` tables, Pydantic schemas with 6-type `BubbleType` Literal + 10-direction `BubbleTailDirection` Literal, 8 CRUD endpoints (Create / Read / Read-list / Update / Delete for both entities), Alembic migration.
- **PDF walker** (WeasyPrint): 7 page-grid templates (`single_panel`, `grid_1x2`, `grid_2x1`, `grid_2x2`, `grid_2x3`, `grid_3x2`, `grid_3x3`), 6 bubble-type CSS variants (speech / thought / narration / shout / whisper / sound_effect), SVG triangle tail primitive with 8 anchor positions + `none` + `auto`. Reuses the 5 KDP picture-book trim sizes.
- **Editor frontend**: `ComicBookEditor` mounts a working multi-panel canvas. `ComicPanelGrid` lays out panels per the page's `comic_panel_grid` template; each panel renders bubbles with positional anchors. Per-panel image upload via `LayoutConfigComicPanel`. Bubble shape preview matches the rendered PDF (BubbleTail SVG math verbatim from the walker).
- **`PdfExportControls` rename** (was `PictureBookPdfExportControls`): 3rd caller surface (PageEditor + BookMetadataEditor + ComicBookEditor) means the picture-book-specific name became a misnomer; `testidPrefix` prop preserves all existing test assertions across the rename.
- **`export_execute` hookspec wired** (`HOOKSPEC-EXPORT-EXECUTE-WIRE-01`): plugin-comics registers `@hookimpl export_execute` gated on `book_type == "comic_book" + fmt == "pdf"`. Eliminates the prior cross-plugin reverse-import in plugin-export's routes.py. Plugin-audiobook retained as documented exception (async + SSE streaming doesn't fit sync `Path | None` hookspec).
- **Comic-book metadata access path** (`COMIC-BOOK-EDITOR-METADATA-BUTTON-01`): the comic-book editor now has the same metadata-button affordance as the prose + picture-book editors.

### KDP Publishing Wizard (Phase 1 + Phase 2)

5-step guided publishing flow for KDP. Largest greenfield frontend feature this cycle.

- **Phase 1 (3 steps)**: MetadataChecklist + CoverValidation + ExportPackage. Calls existing KDP validation endpoints.
- **Phase 2 (2 additional steps)**: PricingStep (royalty calculator with KDP's 35% / 70% split logic) + ArcStep (Advanced Reader Copy reviewer tracking).
- **XState v5 wizard state machine** (`kdpWizardMachine.ts`): replaces the hand-rolled `useState` + step-index pattern. Branching transitions, guards, async side effects gated by state, reset/retry semantics. 20 actor-level tests.
- **Server-side persistence**: `BookPublishingState` schema (single row per book, JSON columns for per-step state) + 5 CRUD endpoints. Wizard auto-saves on each step transition; rehydrates on reopen.
- **ArcReviewer schema** + CRUD: reviewer roster with status (invited / accepted / submitted), one-to-many on book.
- **Conflict-resolution banner**: when `book.updated_at > publishing_state.updated_at` (the user edited the book between wizard sessions), the wizard surfaces a banner offering re-validate-from-scratch vs continue-with-stale-validation.
- **WizardShell composition** (`WIZARD-SHELL-COMPONENT-EXTRACT-01`): both KdpPublishingWizard + ConvertToBookWizard now share `WizardShell` + `WizardNav` primitives. `useState` + step-index logic centralized. ImportWizardModal remains its own pattern (intentional asymmetry — modal vs page-route shape).

### Book-Types Single Source of Truth (BOOK-TYPES-SSOT-YAML-01)

Single canonical declaration of the 3 book types and their capabilities.

- `backend/config/book-types.yaml` — 3 entries (prose, picture_book, comic_book) with per-type capabilities (chapter-based / page-based / KDP-publishable / etc.).
- `BookTypeRegistry` backend — loads YAML at startup with `@lru_cache`; verification gate `test_literal_matches_registry` pins parity with the `Book.book_type` enum.
- `GET /api/book-types` endpoint + frontend `useBookTypes()` hook + `BookTypesProvider` mounted at App root.
- 10 migrated surfaces (capability lookups replace hardcoded book-type lists): `pages.py` PAGEABLE_BOOK_TYPES, Dashboard, GetStarted, CreateBookModal template tab, BookEditor editor dispatch, BookMetadataEditor isChapterBasedBookType, kdp-wizard MetadataChecklist + PricingStep, plugin-getstarted BOOK_TYPES tuple, plugin-kdp/package.py chapter-filter.

### PluginForge v0.7.0 → v0.10.0 adoption

3 successive PluginForge bumps brought a substantial framework upgrade.

- **3-source plugin-metadata pattern codified** in `architecture.md`: class attrs (identity + contract) vs `backend/config/plugins/<name>.yaml` (canonical UI metadata + settings) vs `plugins/<name>/plugin.yaml` (generated ZIP-distribution copy, gitignored). Build-time deduplication via per-plugin Makefile.
- **`target_application` + `app_id` declaration** (`bibliogon` / `bibliogon.app`): plugin identity scoping via PluginForge ≥ 0.7.0 conventions.
- **`min_app_version` moved from YAML to class attribute**: enables version-gating via `pluginforge` constructor's `app_version=__version__` kwarg.
- **DiscoveryResult with severity filtering**: replaces the prior consumer pattern that read PluginManager private state.
- **`/api/admin/rediscover` endpoint**: ad-hoc plugin re-discovery without a server restart (paired with `manager.refresh_config()`).
- **Plugin-status i18n mapping**: `FilterReason` enum values mapped to per-language status strings in 8 catalogs.
- **`/api/settings/plugins/discovered`** extended with per-plugin status + failure-reason for the Settings UI.
- **Single-router-per-plugin convention** finally cleaned plugin-export: 3 top-level routers consolidated to one parent_router with `include_router` nesting (closes the pluginforge 0.8.0 DeprecationWarning).

### Other architecture cleanups

- **`chapter_pre_save` hookspec removed**: dead-intent hook with no implementations.
- **Pages-CRUD router relocated** from plugin-kinderbuch to backend core: comic_book + picture_book share the same CRUD path.
- **User-overlay plugin-enable migration** (`USER-OVERLAY-PLUGIN-ENABLE-MIGRATION-01`): lifespan helper detects new plugins missing from existing user-overlay `plugins.enabled` lists and appends them respecting the user's explicit opt-out signal. Closes the silent-fall-through bug where new plugins (plugin-comics in this release) would fail to activate for upgrade users.
- **Convert-to-book wizard layout stability** (`CONVERT-TO-BOOK-WIZARD-LAYOUT-STABILITY-01`): action-button moved to WizardNav footer + `stepContent` min/max-height. Dialog dimensions stay visually stable across all 6 steps (was: button position jumped between steps).
- **i18n cleanup** (`I18N-ARTICLES-NAMESPACE-CLEANUP-01`): 7 article-editor keys had been silently rendering hardcoded German for non-DE users because they lived under `ui.template_picker.*` while consumers called them via `ui.articles.*`. Fixed across all 8 catalogs.
- **Makefile completion**: all 12 plugins now in `test-plugins` + `test-coverage-plugins` aggregates (was 10 / 9).

## Lessons-learned filed this cycle

- **Single-Router-Per-Plugin convention** (architectural pattern surfaced through halted Comics-Session-2 + restored via refactor).
- **Pre-Coding-Reality-Check: re-audit at the keystroke** (three concrete instances in V060 adoption arc).
- **Audit-Methodology design-intent-axis** (5th-Axis or override-filter — RCU audit hit two design-intent-deferral cases in 48 hours).
- **CRUD shipping: List endpoint is non-optional** (Half-Wired-Lifecycle prevention).
- **Foundation-Override extension** (Half-Wired-Visible-in-Production criterion for P1 promotion).
- **Stale-bundle is the most common false-positive on post-ship user reports** (diagnostic discipline).
- **Plain `git status` before every commit, especially in Multi-Tool-Coordination sessions** (explicit-paths-only staging convention).
- **Periodic backlog re-prioritization discipline** (4-Axes audit pattern).
- **Architecture-doc consultation is part of Pre-Inspection** (single-instance observation from KDP Phase 1).
- **Operational gaps masquerade as wired infrastructure** (mutmut workflow first run).
