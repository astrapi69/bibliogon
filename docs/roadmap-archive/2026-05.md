### DEP-DBPATH-01 step 2 — `BIBLIOGON_DB_PATH` precedence flip

Shipped in v0.28.0. `BIBLIOGON_DATA_DIR` now wins when both env
vars are set; `BIBLIOGON_DB_PATH` is ignored with a clear
"ignored" runtime warning that names the value the user passed.
The DB_PATH-alone path keeps the v0.27.0 deprecation warning so
deployments that still set only the legacy variable migrate at
their own pace. Tests for the prior "both set, no warn" case
were renamed and inverted to assert the new precedence and the
new "ignored" warning. Step 3 (full removal of the
`BIBLIOGON_DB_PATH` override) stays open under the same ID,
queued for the release after v0.28.0. Commit: `7595ee4`.

### D-06 — cross-platform installer scripts

Shipped in v0.28.0. Three new entry points alongside
`install.sh`: `install.command` for macOS (10-line
Finder-doppelklickbar wrapper), `install.ps1` for Windows
(121-line PowerShell mirror generated from
`install.ps1.template` via `make sync-versions`), and
`install.cmd` (7-line Windows batch wrapper invoking
`install.ps1` with `-NoProfile -ExecutionPolicy Bypass` for
Group-Policy-locked corporate environments). All three ship
unsigned per launch decision; macOS Gatekeeper and Windows
SmartScreen warnings documented in both READMEs.
`scripts/sync_versions.py` refactored to a list-driven
`_INSTALL_ARTIFACTS` tuple so `install.sh` and `install.ps1`
share the substitution logic; `verify_version_pins.sh` rejects
hardcoded version literals in the static wrappers
(`install.command`, `install.cmd`). Validation pending on a
fresh macOS user account and fresh Windows 11 install — report
issues with the OS version and exact error message and they
fold into the next point release. Commit: `2b45654`.

### AR-BULK-BOOKS-PARITY-01 — bulk export from the books dashboard

Shipped in v0.28.0. `POST /api/books/bulk-export` accepts
`{book_ids: list[str], format: "epub"|"pdf"|"docx"}`, reuses the
per-book scaffolding pipeline (`_load_book` +
`_scaffold_and_prepare` + `run_pandoc`) per book, and returns a
date-stamped ZIP (`books-YYYY-MM-DD.zip`) of per-book exports.
Hard 200-book limit via Pydantic `max_length`. Filename
collisions get numeric suffixes. Combined-multi-book mode
intentionally not implemented (per-book pipeline produces one
project per book through manuscripta + write-book-template
scaffolding; merging N books would require deciding whose
metadata wins and which book contributes the cover, neither a
natural workflow). Frontend: per-tile checkbox overlay,
Select-all affordance above the grid, sticky `BookBulkActionBar`
with format dropdown + Export + Clear. 8 new
`ui.dashboard.bulk.*` i18n keys in EN + DE backfilled to all
six auto-translated locales. Commits: `ae76e12` (feature),
`d381e64` (CI test fix: pandoc skipif on TestZipShape).

### D-05: Auto-install Docker Desktop — closed as won't-fix

Original ROADMAP P4 entry rescoped on 2026-05-04 (commit
`93af0b2`) from "full Windows installer (Docker Desktop + repo +
`.env`)" to "Docker Desktop auto-install only", because the
Bibliogon launcher already handles repo download, `.env`
generation, Docker image build, and browser open on first run
via `_install_or_welcome` / `_run_install_flow` in
`launcher/bibliogon_launcher/__main__.py`. Only Docker Desktop
itself remained terminal-required for fresh-machine setups.

Closed as **won't fix** on 2026-05-05 after the installer
discovery session
(`docs/explorations/installer-discovery-report.md`) confirmed
that the Docker Subscription Service Agreement Section 2.1's
non-sublicensable license grant forbids third-party silent
install of Docker Desktop. Solo-OSS-dev strict-EULA-reading
default applies; no legal-review budget exists to chase an
ambiguous interpretation.

The launcher's existing **detect-and-instruct** flow (open the
official Docker download page when Docker is missing, continue
the install once Docker is present) is the EULA-compliant
ceiling. Replacing it with a third-party silent install would
violate Docker's terms.

**Citation:** see
[docs/explorations/installer-discovery-report.md](../explorations/installer-discovery-report.md)
section "Docker Desktop integration" for the cited EULA
passages and the analysis chain.

**Related but NOT subsumed:** `LAUNCHER-SELFREPLACE-01` remains
an open P4 backlog item. Discovery confirmed it is a separate
scope (atomic launcher binary self-replace on Windows is
unrelated to Docker Desktop installation).
