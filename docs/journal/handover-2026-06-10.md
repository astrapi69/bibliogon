## Gotchas for whoever touches these next

- **madge runs via `npx madge@8`, NOT a devDependency.** madge's optional
  `typescript@^5.4.4` peer conflicts with the project's TS 6; a pinned devDep
  would break `npm ci` (no `--legacy-peer-deps` per project rule). Keep npx.
- **bandit prints cosmetic `nosec encountered ... no failed test` WARNINGs**
  on the 5 `# nosec B108` lines in `scripts/*umlaut*` — stderr only, exit 0,
  harmless (multi-line argparse line-attribution quirk).
- **dompurify is exact-pinned (`3.4.7`)** and **radix is intentionally not at
  latest** — `npm outdated` will list them. That's expected; see the backlog
  items above before bumping.
- **Two-installation-paths rule:** any shared backend-dep bump must re-lock the
  per-plugin lockfiles (`make lock-all-plugins` / `make verify-plugin-locks`),
  not just `backend/poetry.lock`.
