## Key context / gotchas for next session

- **E2E "passes locally, fails in CI" is the dominant trap this codebase
  has.** Causes seen this session: CI lacks Pandoc (export 500); fresh CI
  data dir triggers onboarding wizards that don't fire locally; CI load makes
  elements resolve-but-not-actionable. When triaging a red E2E gate, pull the
  **CI trace screenshot** (`gh run download <id>`, unzip the spec's
  `trace.zip`, read the latest `resources/*.jpeg`) — looking at the rendered
  frame beat guessing from error text every single time.
- **Rotating failure-set across runs = flake; consistent-set = real.** Compare
  the failing SET run-to-run before treating failures as N independent bugs.
- **`.json → .bgb` format switch (#341):** the Settings full-backup +
  selective-export download `.bgb` ZIPs (with images), not JSON. Don't
  "correct" specs back to `.json`.
- **Parallel sessions share this checkout / one git HEAD.** Verify HEAD before
  committing; stage explicit paths; NEVER `rm`/prune `.claude/worktrees/`
  (it holds another session's live checkout — incident this session, recovered).
  See memory `[[feedback_parallel_session_shared_head]]`.
- **New lessons-learned entries this session** (in `.claude/rules/lessons-learned.md`):
  context.addInitScript-for-multi-tab; E2E-gate-rot-from-un-propagated-format-
  changes + rotating-flake-diagnosis; worktree-rm pitfall.
