## 2. The E2E smoke remediation — the big in-progress work

### 2.1 The discovery
Aster's full `--project=smoke` run showed **85 failures / 347 passed**. Critically, **most failed tests pass in isolation but fail in the serial full run** → systemic **cross-test state pollution**, not 85 individual bugs. Playwright gives each test a fresh page/localStorage, so only **backend state** leaks (DB + `app.yaml` settings).

### 2.2 Root causes (all fixed + pushed in `e44e0cca`)
1. **E2E ran against the developer's REAL data dir** (`~/.local/share/bibliogon`). Every test wipes the DB and mutates `app.yaml` → it was destroying Aster's real data AND leaking settings between tests.
2. **`/api/test/reset` was incomplete** — it deleted only Asset/Chapter/Book/Article, NOT Page/ComicPanel/ComicBubble/StoryEntity/ArticleComment/Publication/etc. → orphan rows accumulated across the serial run.
3. **Settings were never reset** — `ui.dashboard.*_view`, `*_page_size`, `topics` leaked. A test that flipped a view-mode default broke every later card-based test (the app default `articles_view` is **`list`**, but ~all specs drive grid-only `article-card-*` testids).

### 2.3 The systemic fix (committed `e44e0cca`)
- **`e2e/playwright.config.ts`** — the backend webServer now runs ISOLATED: `env: { BIBLIOGON_DATA_DIR=/tmp/bibliogon-e2e-data, BIBLIOGON_TEST=1, TEST_DATABASE_URL=sqlite:////tmp/bibliogon-e2e-data/e2e.db, BIBLIOGON_DEBUG=true }`, `reuseExistingServer: false`, `timeout: 60_000`.
  - `BIBLIOGON_TEST=1` is REQUIRED — it skips the legacy in-tree data-dir migration (`migrate_data_dir_if_needed`), which otherwise fail-loud-conflicts on `backend/plugins/installed` and would try to MOVE the repo's installed-plugins dir. **Gotcha:** `BIBLIOGON_TEST=1` alone forces an in-memory DB; we point `TEST_DATABASE_URL` at a real FILE so data persists across requests within a run.
  - `reuseExistingServer: false` means a stray real-data backend on :8000 is never silently reused. **Run E2E with no other backend on :8000.**
- **`backend/app/main.py`** — `/api/test/reset` now wipes ALL content tables (FK-safe child→parent order, see `_RESET_MODELS_IN_ORDER`). Seed/config tables (BookTemplate*, ChapterTemplate, AudioVoice, GitSyncMapping, Author) are intentionally NOT wiped.
- **`e2e/helpers/api.ts`** — new `resetSettings()`: captures the pre-suite baseline once, then before each test restores the mutation-prone settings (`ui` + `topics`) and **forces both dashboards to grid** (the testid baseline ~all specs assume).
- **`e2e/fixtures/base.ts`** — the auto-fixture now runs `resetDb()` + `resetSettings()` before EVERY test.
- Shared broken-test fixes also in `e44e0cca`: book-card prefix-selector overmatch (it matched `book-card-status-*`; added `:not([data-testid*='-status-'])`) in `dashboard-filters` + `import-wizard(-bgb)`; `trash-view-mode-defaults` stale testids (RadixSelect renders `${testId}-trigger`, the spec used the pre-migration bare ids).

### 2.4 Verified impact
- Isolated backend boots cleanly ("Application startup complete").
- Pollution-prone subset went **18 → 5** failures.
- Full run after the fix: ~91 passed / 5 failed through test ~97; **residual is ~40-50 specs** (down from 85), clustered in the later-alphabet specs.

### 2.5 Earlier individually-fixed specs (committed before `e44e0cca`)

## 4. Critical gotchas (will bite you)

1. **STOP-gate:** do NOT `make release-tag` for v0.45.0 until Aster confirms a clean smoke run. The tag is irreversible.
2. **`; echo EXIT=$?` masks the real exit:** `make X ; echo EXIT=$?` reports the echo's status. Several times `make test`/`release-test` "exit 0" actually FAILED — always grep the output for `FAILED`/`Fehler`/`failed`, don't trust the wrapper exit. This shell is **zsh** → use `${pipestatus[1]}` (not bash `PIPESTATUS`).
3. **Run E2E with no backend on :8000** — `reuseExistingServer: false` will fail to start if the port is taken. Kill stray uvicorns first (`ss -ltnp | grep :8000`; `kill -9 <pid>`). The developer's real `--host` prod server (separate port) is fine to leave.
4. **E2E now uses the isolated `/tmp/bibliogon-e2e-data`** — it no longer touches real data. If you ever revert the isolation, the comprehensive `/test/reset` will destroy the developer's real Page/Comment/StoryEntity rows. Don't.
5. **Default `articles_view` is `list`** (app.yaml) but specs assume grid — `resetSettings()` forces grid; list-specific specs set their own view AFTER the reset.
6. **RadixSelect testids:** trigger is `${testId}-trigger`, options are `${testId}-item-${value}` (and `-item-all`). A static grep for the bare `testId` won't find them (templated) — they DO exist at runtime.
7. **A flexbox child's `inline-flex` is blockified to `flex`** in `getComputedStyle` — don't assert `display: inline-flex` on a `.btn` inside a flex container; assert the classes.
8. **happy-dom/jsdom can't see CSS/layout** — a passing component test does NOT prove visibility (the Verlauf modal had passing tests but undefined CSS made it invisible). UI-visibility needs Playwright.
9. **Foreground `sleep` is blocked** in the Bash tool; background runs are harness-tracked (you're re-invoked on completion) — don't poll-sleep.
10. **German content uses real umlauts; chat stays ASCII.** i18n default test locale is `de` (untouched by the es/fr/el/pt/tr/ja sweep).
