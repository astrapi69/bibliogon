## Adjudication recap

Two mid-arc decisions worth noting for future sessions:

1. **F1 — "keep text-styling CSS only".** Followed the literal directive: trimmed to `.soundEffect` text rules. The `.soundEffect` class isn't currently applied at runtime (the `ComicBubble.tsx` text-overlay div uses inline styles, not class names) — so the kept CSS is functionally orphaned today. Wiring it up would change runtime behavior; deferred as a separate enhancement if the sound-effect typography becomes a real authoring need.

2. **Shape swap (`884fc7f`).** Triggered by user-confirmed visual evidence after two messages of status-correction discipline. The first two requests to "swap" were resisted with concrete evidence the current code already matched the stated target. The third request — with explicit user confirmation of a live-running mismatch — was executed without further pushback. The pattern matches the [Multi-tool collaboration tracking](../../.claude/rules/lessons-learned.md) lessons-learned rule: status correction is load-bearing, then the user's reconfirmation is authoritative.

Would be a separate cleanup commit.

- `docs/audits/comic-bubble-konzept.md` — the concept doc that drove the original Path B work. Predates the swap, so its text describes speech=ellipse and thought=rounded-rect. Stays untouched: it's a frozen design artefact, not active documentation. The help docs in `docs/help/*/books/comic-bubbles.md` are the live source of truth.
